⚜️ Aurum — Functional & Operations Guide

🏦 What Is Aurum?

Aurum is a self-hosted AI-powered financial education assistant. Users can chat about investing, their portfolio, market data, tax strategies, and personal finance goals. It combines a curated knowledge base (RAG) with real-time market data and a multi-agent LangGraph pipeline to route each question to the right specialized agent.

🤖 AI Chat
Multi-agent LangGraph pipeline with supervisor routing, RAG retrieval, and streaming responses.
📊 Portfolio
Track holdings, view allocation, monitor gain/loss with manual entry or import.
📈 Market Data
Live quotes, index prices, and news via Alpha Vantage / Yahoo Finance.
🎯 Goals
Set and track savings goals (retirement, house, emergency fund, college).
📰 News
Financial news feed with AI-powered summaries and sentiment tagging.
🛡️ Admin
Token usage dashboard, per-message traces, user analytics, LangSmith integration.
Disclaimer notice: Aurum is an educational tool. All responses include a disclaimer that the content does not constitute financial advice. This text is configurable in config.yaml.

🔄 Chat Data Flow

What happens end-to-end when a user sends a message:

1
WebSocket connects — browser opens ws://localhost:8000/api/chat/ws?conversation_id=…&token=…. JWT is validated on connect; if invalid, the connection is rejected.
2
Message received — raw text arrives. A HumanMessage is wrapped and passed to LangGraph with thread_id = conversation_id.
3
Supervisor routes — supervisor_route calls the LLM to decide which agents to invoke (qa, portfolio, market, goals, news, tax) and which RAG categories to search. Returns intent, routing_reason, and confidence.
4
RAG retrieval — retrieve_rag queries ChromaDB with the user's message, optionally filtered to the chosen categories. Returns top-5 chunks by cosine similarity.
5
Agents run — selected agents run in parallel (or sequentially depending on routing). Each agent has its own system prompt and uses the RAG context + conversation history. on_chat_model_stream events stream tokens to the browser in real time.
6
Synthesizer — combines agent outputs into a single coherent response, adds citations for any RAG sources used.
7
Persist — persist_turn saves the assistant message to Postgres. A ChatTrace record is written with: intent, routing decision, RAG docs retrieved, per-agent token counts and latency, supervisor confidence.
8
Final frame — a {"type":"final"} WebSocket frame is sent with message_id, full content, agents_used, and citations list.

WebSocket Frame Types

TypePayloadWhen
agent_start{agent: "tax"}Agent node begins
agent_end{agent: "tax"}Agent node finishes
token{delta: "..."}Each streaming token
citation{source_title, source_url, category}Per RAG source used
final{message_id, content, agents_used, citations}Stream complete
error{message: "..."}On exception

🚀 Quick Start

Prerequisites

  • Docker Desktop (or Docker Engine + Compose v2)
  • An LLM API key (OpenAI, Anthropic, Google, or AWS Bedrock)
  • ~4 GB free disk space for images + data volumes

First-Time Setup

# 1. Copy and edit environment file
cp .env.example .env

# 2. Start all services
docker compose up -d

# 3. Run DB migrations
docker compose exec api alembic upgrade head

# 4. Seed RAG knowledge base
docker compose --profile seed run --rm rag-ingest

# 5. Open the app
open http://localhost:5173

Day-to-Day Commands

# Start all services
docker compose up -d

# Stop all services
docker compose down

# View logs (all services)
docker compose logs -f

# View logs (API only)
docker compose logs -f api

# Restart API after code change
docker compose restart api
# (or just edit a file — uvicorn --reload detects changes automatically)

# Run DB migration after schema change
docker compose exec api alembic upgrade head

# Open Postgres shell
docker compose exec postgres psql -U aurum -d aurum

# Check ChromaDB health
curl http://localhost:8001/api/v2/heartbeat

Nuclear Reset

This destroys all data: users, conversations, portfolio, and the RAG vector store. 
docker compose down -v   # removes named volumes
docker compose up -d
docker compose exec api alembic upgrade head
docker compose --profile seed run --rm rag-ingest

📚 How RAG Works

RAG (Retrieval-Augmented Generation) lets the AI answer questions using your specific knowledge base rather than relying solely on what the model was trained on.

1
Ingest — Markdown files are parsed, chunked into ~800-word blocks with 100-word overlap, and stored in ChromaDB as vector embeddings alongside their metadata.
2
Route — The supervisor decides which categories are relevant to the query (e.g. a tax question → ["tax"]). This narrows the search space.
3
Retrieve — ChromaDB runs a cosine similarity search against the query, filtered to the selected categories, returning the top 5 chunks.
4
Inject — Retrieved chunks are formatted and injected into the agent's context as Source [N]: … blocks. The model cites sources in its response.
5
Cite — Any source_title and source_url from retrieved docs appear as citation links below the response in the chat UI.
source_url is citation metadata only — it is NOT scraped. The ingest script reads only the markdown file's text content. If you add a source_url, it appears as a clickable reference link in responses but the URL's content is not fetched or indexed. To index a web page, you must copy its text into a .md file yourself.

ChromaDB Embedding

By default ChromaDB uses its built-in all-MiniLM-L6-v2 embedding model (runs locally, no API call). The collection uses cosine distance for similarity ranking. The collection name is finance_kb.

➕ Adding Documents to the Knowledge Base

Step 1 — Create the Markdown file

Place the file in the correct category subfolder under src/rag/seed/:

src/rag/seed/
  investing/     ← for investing concepts
  portfolio/     ← for portfolio management
  tax/           ← for tax strategies
  goals/         ← for savings goals
  market/        ← for market mechanics

The file format is Markdown with YAML frontmatter:

---
source_title: How to Read an Earnings Report
category: market
source_url: https://investor.gov/...   # shown as citation link — NOT scraped
---

# How to Read an Earnings Report

An earnings report (Form 10-Q or 10-K) is filed quarterly or annually...

## Revenue and Net Income
...

## Key Ratios to Watch
- **EPS** (Earnings Per Share): net income divided by shares outstanding
- **P/E Ratio**: price divided by EPS — how much investors pay per dollar of earnings
...
Frontmatter fields:
  • source_title — displayed as the citation label in chat responses. Falls back to the filename if omitted.
  • category — should match the folder name. Used for category-filtered retrieval.
  • source_url — optional URL shown as a clickable link in citations. Leave empty if no external source.

Step 2 — Run ingest

# Additive — adds new docs, keeps existing ones
docker compose exec api python -m src.rag.ingest --seed-dir src/rag/seed

# Full rebuild — clears all docs then re-ingests everything
docker compose exec api python -m src.rag.ingest --seed-dir src/rag/seed --clear

# Or using the docker compose profile (equivalent)
docker compose --profile seed run --rm rag-ingest

Step 3 — Verify

# Check total chunk count in ChromaDB
docker compose exec api python -c "
from src.rag.chroma_store import ChromaStore
s = ChromaStore()
print('Total chunks:', s.count())
"

Chunking Behavior

Each document is split into overlapping chunks before storage:

ParameterDefaultDescription
CHUNK_SIZE800 wordsMaximum words per chunk
CHUNK_OVERLAP100 wordsWords shared between adjacent chunks

A 1,200-word article produces 2 chunks. A 3,000-word article produces ~5 chunks. Each chunk gets a SHA256-based ID derived from filepath:chunk_index, so re-ingesting the same file is idempotent (upsert — no duplicates).

🗂️ Categories & Routing

Categories are the bridge between user intent and the right knowledge. The supervisor LLM assigns one or more categories to each query; the RAG retriever filters ChromaDB to only those categories before running the similarity search.

CategoryAgentExample Queries
investing qa "What is dollar-cost averaging?" · "How do index funds work?"
portfolio portfolio "How diversified am I?" · "Should I rebalance?"
market market "What's the S&P 500 at?" · "Explain P/E ratios"
goals goals "Help me plan for retirement" · "How much for a house down payment?"
tax tax "How are capital gains taxed?" · "What's a Roth IRA?"

Adding a New Category

If you want to add a new topic area (e.g. crypto or options):

  1. Create the folder: src/rag/seed/crypto/
  2. Add .md files with category: crypto in frontmatter
  3. Add "crypto" to the CATEGORIES tuple in src/rag/chroma_store.py:
CATEGORIES = ("investing", "portfolio", "tax", "goals", "market", "crypto")
  1. Re-run ingest: docker compose exec api python -m src.rag.ingest --seed-dir src/rag/seed
  2. The supervisor will start routing crypto questions to the new category automatically (it reads the category list and uses it in its routing prompt).
You don't need to create a new agent for a new category — existing agents (especially qa) will handle the retrieved context. Create a dedicated agent only if you need specialized tool calls or a very different system prompt.

⚡ Running Ingest

Methods

MethodCommandUse When
Via exec (live container) docker compose exec api python -m src.rag.ingest --seed-dir src/rag/seed Adding new docs to a running instance
Via profile (one-shot) docker compose --profile seed run --rm rag-ingest First-time setup or CI/CD
With clear flag … python -m src.rag.ingest --seed-dir src/rag/seed --clear Full rebuild — e.g. after renaming files or changing chunk size
Custom directory … --seed-dir /path/to/my/docs Ingesting from a different folder

What Ingest Does

  1. Scans all *.md files recursively under the seed directory
  2. Parses YAML frontmatter for source_title, category, source_url
  3. Splits content into word-count chunks (800 words, 100 overlap)
  4. Generates a deterministic SHA256 ID per chunk: sha256("filepath:chunk_idx")[:16]
  5. Calls collection.upsert() — safe to run multiple times, won't create duplicates

Expected Output

  investing/what-is-investing.md → 2 chunk(s)
  investing/dollar-cost-averaging.md → 1 chunk(s)
  ...
  tax/tax-loss-harvesting.md → 2 chunk(s)

🗄️ Managing ChromaDB Documents

Delete Specific Documents

docker compose exec api python3 - <<'EOF'
from src.rag.chroma_store import ChromaStore
s = ChromaStore()

# Delete all chunks from one category
s._col.delete(where={"category": "tax"})
print("After delete:", s.count())
EOF
docker compose exec api python3 - <<'EOF'
from src.rag.chroma_store import ChromaStore
s = ChromaStore()

# Delete all chunks from a specific source title
s._col.delete(where={"source_title": "Capital Gains Tax Explained"})
EOF

Replace a Single Document

Since ingest uses upsert with deterministic IDs, just edit the .md file and re-run ingest — the old chunks for that file will be overwritten automatically (same SHA256 IDs).

If you rename a file, the old chunks (keyed to the old path) will remain in the store and the new file will add new chunks. Use --clear after renaming to avoid stale data.

🔧 Troubleshooting

Chat returns an error immediately

docker compose logs api --tail=30

Common causes:

  • Invalid or missing LLM_API_KEY → check .env and restart API
  • LLM rate limit exceeded → wait and retry
  • ChromaDB unreachable → docker compose ps chroma should show healthy

RAG returns no results

# Check chunk count
docker compose exec api python -c "from src.rag.chroma_store import ChromaStore; print(ChromaStore().count())"

If count is 0, ingest hasn't been run. Run:

docker compose --profile seed run --rm rag-ingest

Migrations out of date

docker compose exec api alembic upgrade head

If you see Table 'chat_traces' doesn't exist or similar, migrations haven't been applied.

Admin nav item not showing

Check that your email exactly matches ADMIN_EMAILS in .env (case-insensitive, but no extra spaces). Restart the API after changing the env file.

ChromaDB version mismatch

The Python client version must match the server version. The current setup uses chromadb/chroma:latest in docker-compose and the latest Python client. If you see KeyError('_type') on collection create, the versions are mismatched — update docker-compose.yml to chromadb/chroma:latest.
