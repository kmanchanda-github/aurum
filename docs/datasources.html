⚜️ Aurum — Data Sources & Tab Reference

🗂️ All Tabs at a Glance

Each tab in Aurum pulls from a different combination of live external APIs, local Postgres storage, AI inference, and a built-in knowledge base. This document explains exactly what each tab does and where its data comes from.

⚡ Caching Layer

All live API responses are cached in Redis to minimise external API calls and keep UI response times fast. Cache is per-symbol or per-query, keyed by a SHA256 hash.

Data TypeTTLConfig KeyRationale
Stock quote60 secondsCACHE_TTL_QUOTENear-real-time for chat agent use
Price history5 minutesCACHE_TTL_HISTORYCharts don't need tick-level freshness
Index prices1 hourCACHE_TTL_INDEXDashboard overview, refresh on page load is fine
News feed15 minutesCACHE_TTL_NEWSGoogle News RSS rate limits; content doesn't change per-minute

Override any TTL by setting the corresponding env var in .env. To disable Redis and use an in-process cache instead (e.g. HF Spaces), set USE_IN_MEMORY_CACHE=true.

💬 Chat Tab

The most complex tab. Every message runs through a LangGraph multi-agent pipeline. Three separate data sources contribute to each response.

Source 1 — LLM Inference AI Inference

What: Every agent node in the graph calls the configured LLM to generate its response. The supervisor also calls the LLM to decide routing.

Provider: Set by LLM_PROVIDER + LLM_MODEL in .env. Supported: OpenAI, Anthropic, Google, AWS Bedrock.

Cost: Billed per token by the provider. Visible in Admin → Conversations trace.

No caching — every LLM call goes to the provider API.

Source 2 — RAG Knowledge Base Knowledge Base

What: 21 pre-written educational articles ingested into ChromaDB as vector embeddings. Retrieved by cosine similarity on the user's query, filtered to the categories the supervisor selects.

Where the content came from: Written as original educational content when the project was built — general personal finance concepts (DCA, index funds, capital gains, Roth IRA, etc.). No external URLs were scraped. All source_url fields are empty.

Collection: finance_kb in ChromaDB. Embedding model: ChromaDB default (all-MiniLM-L6-v2, runs locally).

How to add content: See the Operations Guide → Adding Documents.

Source 3 — Live Market Data Live API

What: The market_agent calls registry.get_quote() for real-time prices when the user's query involves current market data.

Provider: yfinance (primary). Alpha Vantage if configured and yfinance fails.

Cache: 60-second Redis TTL per symbol.

Agent Pipeline

User message → Supervisor (LLM) → routes to → RAG retrieval + Agent(s) (LLM) → Synthesizer (LLM) → Persist to DB

📈 Market Tab

Real-time and historical price data for stocks, ETFs, and market indices. Uses an adapter registry so you can add or swap data providers without changing the API routes.

Primary Source — yfinance Live API

Library: yfinance — an unofficial Python wrapper around Yahoo Finance's data endpoints.

API key required: No. Free to use, no registration.

Data available: Real-time quotes (15-min delayed for most exchanges), full OHLCV history (1d to max), 52-week high/low, market cap, 3-month average volume, symbol search.

Limitations: Unofficial API — Yahoo Finance can change endpoints without notice. Rate limits are not published but throttling can occur under heavy load. Not suitable for high-frequency trading data.

Config: Enabled by default in config.yaml. Priority: 10 (lowest number = highest priority).

Secondary Source — Alpha Vantage Live API

API key required: Yes — ALPHA_VANTAGE_API_KEY in .env. Free tier: 25 requests/day. Premium tiers available.

Data available: Real-time quotes, daily/weekly OHLCV history, fundamental data.

Status: Disabled by default (enabled: false in config.yaml). Used as fallback if yfinance fails and the key is set.

Enable it: Set ALPHA_VANTAGE_API_KEY=your_key in .env and flip alpha_vantage.enabled: true in config.yaml.

Free tier sign-up: alphavantage.co

Configured Indices

The indices shown on the market overview are set in config.yaml → indices:

Add or remove indices by editing the indices list in config.yaml and restarting the API. Any symbol supported by yfinance works.

Market Movers

The movers endpoint (GET /api/market/movers) does not use a real gainers/losers screener. It fetches live quotes for a hardcoded list of 8 popular symbols (AAPL, MSFT, NVDA, GOOGL, AMZN, META, TSLA, JPM) and sorts them by change_pct. This is a placeholder — true market-wide screeners require a premium data subscription (e.g. Polygon.io, Finnhub, or Alpha Vantage premium).

📊 Portfolio Tab

Tracks the user's investment holdings. All data is manually entered — there is no brokerage integration or automatic import.

Data Source — PostgreSQL Database

Tables: portfolios (one per user, named), holdings (symbol, quantity, cost basis per portfolio).

What you enter: Ticker symbol, number of shares/units, cost basis (average purchase price). Optionally: asset class label.

Live prices: The portfolio detail endpoint attempts to enrich holdings with live prices, but the current implementation returns cost-basis totals only (live price enrichment is a pending enhancement). The portfolio_agent in Chat does call yfinance for live prices when analysing your holdings via AI.

No External Brokerage Integration

Aurum does not connect to Robinhood, Schwab, Fidelity, or any other brokerage. To add holdings, enter them manually on the Portfolio tab. If you want brokerage import in the future, the architecture supports it via new adapter in src/adapters/.

What the Portfolio Agent Sees (in Chat)

When you ask a portfolio question in Chat, the portfolio_agent reads your holdings from Postgres, then calls yfinance to get current prices, calculates gain/loss, and passes this context to the LLM. The LLM response is AI-generated analysis, not a guaranteed calculation.

🎯 Goals Tab

Savings goal tracker with Monte Carlo projection. No external data — entirely self-contained.

Data Source — PostgreSQL Database

Table: goals — stores goal name, type, target amount, current amount, monthly contribution, target date, and risk tolerance.

Goal types supported: retirement, house, emergency fund, college, custom.

Projection Engine — Monte Carlo Simulation Computed

What it does: Runs 1,000 simulated paths of your savings over a chosen time horizon using randomly sampled monthly returns, then reports p10/p50/p90 percentiles and a probability-of-success score.

Library: NumPy (np.random.default_rng). No external API.

Assumptions baked into the code — these are not sourced from any external model:

Risk ProfileAnnual ReturnAnnual VolatilityBased on
Conservative5%8%Bond-heavy portfolio approximation
Moderate7%12%60/40 stock-bond blend approximation
Aggressive10%18%Equity-heavy portfolio approximation
These return/volatility assumptions are hardcoded estimates, not market forecasts. They do not update with market conditions. Real financial planning tools use forward-looking models or historical calibration. Treat projections as illustrative only.

📰 News Tab

Financial news feed powered by Google News RSS.

Source — Google News RSS Live API

Library: pygooglenews — a Python wrapper around the Google News RSS feed. Falls back to direct feedparser if the library fails.

API key required: No. Google News RSS is publicly accessible.

What it returns: Article titles, URLs, publication timestamps, source name (e.g. "Reuters", "CNBC", "WSJ"), and a short summary snippet.

Important: Aurum receives metadata from Google's aggregator — not the full article text. Clicking a headline opens the original publisher's page. Google News does not publish which outlets it indexes; coverage varies.

Language/Region: English / United States (configurable in config.yaml → news_sources.pygooglenews).

Cache: 15 minutes (Redis). The same query will not re-hit Google News within that window.

Default Search Query

If no ?query= param is passed, the API defaults to "financial markets investing". The News tab UI passes this default. You can search for specific topics (e.g. "Apple earnings", "Federal Reserve interest rates") by changing the query parameter.

Article Limit

Up to 30 articles per request (default: 10). Set via ?limit=N on GET /api/news.

Content Provenance

Articles are aggregated by Google from thousands of publishers. Aurum does not verify, curate, or filter content beyond what Google News returns. Source quality depends entirely on what Google's algorithm surfaces for the given query.

Custom queries: The News tab currently uses the default query for all users. To support per-user topic filtering (e.g. follow specific tickers), pass ?query=AAPL+earnings to GET /api/news and update the UI to expose a search box.

GET /api/settings/adapters/health calls health_check() on each registered market adapter and returns response latency in milliseconds. Useful for diagnosing whether yfinance or Alpha Vantage is reachable.

Tax Explanation Endpoint AI Inference

Endpoint: POST /api/tax/explain

What it does: Calls the LLM directly (not via LangGraph) with a focused educational system prompt. Returns a 3–5 paragraph explanation plus the standard disclaimer.

Temperature: 0.2 (lower than the main chat pipeline) for more deterministic, factual responses.

Not backed by RAG — this endpoint does not query ChromaDB. For RAG-grounded tax answers, use the main Chat tab.

➕ Adding a Market Data Source

The adapter registry pattern makes it straightforward to add new providers.

Step 1 — Create the adapter

# src/adapters/market/mybroker_adapter.py
from src.adapters.base import MarketDataAdapter, Quote

class MyBrokerAdapter(MarketDataAdapter):
    name = "mybroker"
    capabilities = {"quote", "history"}

    async def get_quote(self, symbol: str) -> Quote:
        # ... call your API ...
        return Quote(symbol=symbol, price=..., ...)

    async def get_history(self, symbol, period, interval):
        # ...

Step 2 — Register it in config.yaml

data_sources:
  mybroker:
    enabled: true
    priority: 5          # lower = higher priority than yfinance (10)
    api_key: ${MYBROKER_API_KEY}

Step 3 — Wire it in registry.py

# In build_registry() in src/adapters/registry.py
if ds.get("mybroker", {}).get("enabled"):
    from src.adapters.market.mybroker_adapter import MyBrokerAdapter
    registry.register_market(MyBrokerAdapter(), priority=ds["mybroker"]["priority"])

The registry automatically falls through to the next adapter if the primary one raises an exception.

    async def fetch(self, query, since=None, limit=10) -> list[NewsItem]:
        # call https://newsapi.org/v2/everything?q=query&apiKey=...
        ...

Step 2 — Add to config.yaml

news_sources:
  pygooglenews:
    enabled: true
    language: en
    country: US
  newsapi:
    enabled: true          # will run in parallel / as fallback

Step 3 — Register in registry.py
