Project Overview

📌 What It Is

Aurum is a production-ready, multi-agent AI system for financial literacy. Users interact through a streaming chat interface backed by six parallel specialist agents — each grounded in a private RAG corpus and real-time market data — orchestrated by a LangGraph supervisor.

🎯 Core Value Proposition

  • Six specialist agents run in parallel via Send-fanout
  • Typed WebSocket frames for real-time streaming UX
  • RAG over 21 curated financial education articles
  • Pluggable LLM providers — swap without code changes
  • Pluggable market data adapters with registry fallback

📚 Course Context

  • Course: Applied Agentic AI for SWEs
  • Format: 7-day capstone sprint
  • Goal: end-to-end production AI system
  • Focus: LangGraph, RAG, streaming, adapters
  • Deploy: local Docker + Hugging Face Spaces

⚙️ Tech Stack at a Glance

  • FastAPI + LangGraph + Celery
  • PostgreSQL + Redis + ChromaDB
  • React 18 + Vite + TypeScript + Tailwind
  • Recharts for data visualizations
  • Docker Compose + nginx (production)

Risk Register

Risk Likelihood Impact Mitigation
Google Finance API unavailable or rate-limited High High Use yfinance (Yahoo Finance data, identical coverage) as primary; AlphaVantage as registered fallback
LLM API costs exceed budget during development Medium Medium Configurable provider via .env; use Claude Haiku for supervisor routing, full model only for synthesis
Hugging Face Spaces resource/memory limits Medium Low SQLite + in-memory cache fallback in Dockerfile.hf; single-container architecture avoids multi-service orchestration
RAG retrieval quality insufficient for financial queries Low Medium 21 curated seed articles + query-time category filtering in ChromaDB; hybrid keyword + semantic search
LangGraph parallel agent race conditions on shared state Low High Annotated[list, add] reducer on agent_results field; no shared mutable state between agents

Deployment Targets

🐳 Local Development dev

All services on a Docker bridge network. Postgres, Redis, API, and Celery workers start with a single command.

docker compose up
  • Postgres 15 on port 5432
  • Redis 7 on port 6379
  • FastAPI + Uvicorn on port 8000
  • Celery worker (4 concurrency)
  • Hot-reload via volume mount

🚀 Production prod

nginx reverse proxy in front of 4 Uvicorn workers. SSL termination at nginx, static files served directly.

docker compose -f docker-compose.prod.yml up
  • nginx on ports 80/443
  • 4 Uvicorn workers behind nginx
  • React SPA served as static files
  • Managed Postgres + Redis (external)

🤗 Hugging Face Spaces hf

Single container on port 7860. SQLite replaces Postgres; in-memory dict replaces Redis for zero-dependency deploy.

Dockerfile.hf — port 7860
  • SQLite persistence (no external DB)
  • In-memory cache fallback
  • README.md HF metadata block
  • Zero external service dependencies
