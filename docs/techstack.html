LangGraph

Multi-agent orchestration with stateful graphs

🔀

LangGraph

v0.2.50+ · Python
Stateful agent graph framework by LangChain

LangGraph lets you define agents as nodes in a directed graph. Each node is a Python function that reads from and writes to a shared State TypedDict. Edges can be conditional (router decides which node runs next) or use Send for parallel fan-out.

Why LangGraph over plain LangChain?

LangChain chains are linear and stateless — good for single-turn pipelines. LangGraph adds cyclic graphs, persistent checkpoints, and parallel execution. Aurum needs all three: a supervisor that routes to multiple specialists in parallel, conversation memory across turns, and branching logic (e.g., route to "clarify" if confidence is low).

# Parallel fan-out with Send()
def fanout_to_agents(state: FinanceState) -> list[Send]:
    return [
        Send(agent_name, {"messages": state["messages"], "user_id": state["user_id"], ...})

LangChain

LLM primitives, message types, and tool infrastructure

⛓

LangChain Core

langchain>=0.3.9, langchain-core>=0.3.21
Message types, prompt templates, and LLM interfaces

LangChain provides the shared vocabulary that all agents and LLM providers speak: HumanMessage, AIMessage, SystemMessage, ChatPromptTemplate. This lets you swap LLM providers without touching agent code.

What Aurum uses it for

Message objects in FinanceState.messages, prompt templates for the 6 agent nodes, astream_events for streaming token-by-token to the WebSocket, and add_messages as the LangGraph state reducer that correctly merges message lists.

# Agent node pattern
async def qa_agent(state: FinanceState) -> dict:
    llm = get_llm(temperature=0.3, streaming=True)
    prompt = ChatPromptTemplate.from_messages([
        ("system", QA_SYSTEM_PROMPT),
        MessagesPlaceholder("messages"),
    ])
    chain = prompt | llm
    response = await chain.ainvoke({"messages": state["messages"], ...})
    return {"agent_results": [AgentResult(agent="qa", content=response.content)]}

Multi-Provider LLM Factory

src/core/llm.py — configure via .env

Aurum's get_llm() factory returns a LangChain-compatible chat model regardless of provider. All agent code calls get_llm() — no provider-specific imports anywhere except in llm.py.

Why abstract the provider?

ChromaDB

Vector store for RAG — semantic search over financial articles

🗄

ChromaDB

v1.5.x client + latest server
Open-source embedding database

ChromaDB stores document embeddings (dense float vectors) and lets you query by semantic similarity. When a user asks a financial question, Aurum embeds the query and finds the most relevant knowledge-base articles — this is RAG (Retrieval-Augmented Generation).

Why ChromaDB over Pinecone, Weaviate, or pgvector?

ChromaDB runs fully locally with a single Docker image — no API key, no cloud cost, no signup. Pinecone and Weaviate Cloud both require external API keys. pgvector would work but adds schema complexity to PostgreSQL. For a capstone where portability matters, a self-contained Docker service wins.

# Category-filtered semantic search
def query(self, text: str, categories: list[str], k: int = 5):
    where = {"category": {"$in": categories}} if len(categories) > 1 \
        else {"category": categories[0]}
    return self._col.query(
        query_texts=[text],
        n_results=k,
        where=where,  # metadata filter
        include=["documents", "metadatas", "distances"],
    )

# Collection uses cosine similarity (better for text than L2)
self._col = self._client.get_or_create_collection(
    name="finance_kb",
    metadata={"hnsw:space": "cosine"},
)

The 51 seed articles are split across 5 categories (investing, portfolio, tax, goals, market). The supervisor selects relevant categories, so a portfolio question only retrieves portfolio articles — reducing noise.

Embeddings

Turning text into vectors — the math behind semantic search

🔢

all-MiniLM-L6-v2

sentence-transformers>=3.3.0 · 384 dimensions
Lightweight BERT-based sentence embedding model

An embedding model converts text into a fixed-length float vector. Sentences with similar meaning end up near each other in vector space — enabling semantic (meaning-based) search instead of keyword matching.

Why all-MiniLM-L6-v2?

It's the default ChromaDB embedding function — runs locally (no API call), fast (6 transformer layers), and produces 384-dim vectors that are small enough to be efficient while accurate enough for finance Q&A. OpenAI's text-embedding-3-small is slightly better quality but costs money per embedding call and requires internet access at ingest time.

How HNSW works: Hierarchical Navigable Small World is the approximate nearest neighbor index ChromaDB uses internally. It builds a graph of vectors at multiple "zoom levels" — like a geographic map — letting you find the ~nearest vectors in O(log n) time instead of brute-force O(n).

FastAPI

Async Python web framework — the API gateway

⚡

FastAPI

>=0.115.0 + uvicorn[standard]
Modern async Python API framework

FastAPI generates OpenAPI docs automatically from Python type hints. Routes are async by default, meaning a single process can handle thousands of concurrent requests without threads — essential when each chat request may take several seconds awaiting an LLM.

Why FastAPI over Flask / Django?

Flask and Django REST Framework are synchronous by default. Aurum's LangGraph astream_events requires an async event loop throughout the entire request lifecycle. FastAPI + uvicorn is the standard async Python stack, and pydantic-v2 integration means request/response validation is free.

) -> QuoteResponse:

WebSockets + Streaming

Real-time token streaming from LLM → browser

📡

WebSocket Streaming

FastAPI WS + LangGraph astream_events
Full-duplex streaming with typed JSON frames

Instead of waiting for the full LLM response (5–15 seconds), tokens stream to the browser as they're generated. FastAPI's WebSocket endpoint stays open for the session lifetime. The React useChat hook reconnects automatically with exponential backoff.

Why WebSockets over SSE?

Server-Sent Events (SSE) only support server→client streaming over HTTP. WebSockets are full-duplex, allowing the client to send new messages while prior responses are still streaming, and enabling mid-stream cancellation. For a chat app, WS is the natural choice.

SQLAlchemy

Async ORM for users, portfolios, goals, and conversations

🗃

SQLAlchemy 2.0 (async)

>=2.0.36 + asyncpg driver
Python ORM with async session support

SQLAlchemy 2.0 added first-class async support via AsyncSession. Combined with the asyncpg driver, all DB queries are non-blocking — the event loop can handle other requests while waiting for Postgres.

Why SQLAlchemy over raw SQL or Tortoise ORM?

Raw SQL requires manual parameter escaping and schema management. Tortoise ORM has a smaller ecosystem. SQLAlchemy is the de facto standard with excellent Alembic migration support, lazy loading, relationship support, and battle-tested async sessions. Pydantic models validate at the API layer; SQLAlchemy models handle persistence.

JWT Auth & bcrypt

Stateless authentication and password hashing

🔐

python-jose + bcrypt

python-jose[cryptography]>=3.3.0 + bcrypt>=4.0.0
JWT tokens + adaptive password hashing

Passwords are hashed with bcrypt (adaptive, salted, intentionally slow). On login, a JWT is issued — a cryptographically signed JSON payload containing the user ID and expiry. Every subsequent request carries this token; the server verifies the signature without a DB lookup.

Why JWT over sessions?

Session-based auth requires a shared session store (Redis, DB) — every service instance must hit it on each request. JWTs are self-contained: the server just verifies the HMAC signature. This makes horizontal scaling trivial — no sticky sessions, no shared cache for auth. The trade-off is tokens can't be revoked before expiry (acceptable for a 7-day TTL with refresh).

# Password: one-way hash, different salt each time
def hash_password(password: str) -> str:
    return bcrypt.hashpw(password.encode(), bcrypt.gensalt()).decode()

# JWT: HS256 signed with SECRET_KEY
# Header.Payload.Signature — base64url encoded
payload = {"sub": user_id, "exp": now + timedelta(days=7)}
token = jwt.encode(payload, SECRET_KEY, algorithm="HS256")

# Verify in FastAPI dependency
async def get_current_user(token: str = Depends(oauth2_scheme)):
    payload = decode_token(token)  # raises ValueError if invalid
    user = await db.get(User, payload["sub"])

Rate Limiting

slowapi — preventing abuse on public endpoints

🚦

slowapi

>=0.1.9
Flask-Limiter port for FastAPI / Starlette

slowapi wraps limits and integrates with FastAPI's request lifecycle. Rate limit state is tracked per IP address. When the limit is hit, it raises an exception that FastAPI's exception handler converts to a 429 response.

The circular import trap — and how we solved it

Both main.py (adds the limiter to app state) and auth.py (uses @limiter.limit()) need the same Limiter instance. If auth.py imports from main.py and main.py imports auth.py, Python raises a circular import error. Solution: extract limiter to its own module src/api/limiter.py — both files import from there.

# src/api/limiter.py — breaks the circular import
from slowapi import Limiter
from slowapi.util import get_remote_address
limiter = Limiter(key_func=get_remote_address)

# auth.py — uses the shared instance
from src.api.limiter import limiter

@router.post("/register")
@limiter.limit("10/minute")  # 10 registrations per IP per min
async def register(request: Request, ...): ...

PostgreSQL

Primary relational database for all structured data

🐘

PostgreSQL 16

postgres:16-alpine Docker image
The world's most advanced open-source relational database

PostgreSQL stores users, portfolios, holdings, goals, conversation history, and LangGraph checkpoints. The -alpine image keeps the container tiny (≈85MB). JSONB columns store flexible LangGraph checkpoint data without needing a schema migration for every state change.

Why Postgres over SQLite / MySQL?

SQLite is single-writer only — with multiple uvicorn workers, concurrent writes would lock. MySQL lacks some JSONB operators Aurum uses for checkpoint storage. Postgres is the production standard for Python web apps, has excellent async driver support (asyncpg), and the same Docker image runs locally and in most cloud environments.

Schema tables: users user_settings portfolios holdings goals conversations messages

Redis

In-memory cache for market data and API responses

⚡

Redis 7

redis:7-alpine + redis-py>=5.2.0
In-memory data store used as a cache

Market quotes and news headlines are cached in Redis with TTLs (60s for quotes, 5 minutes for indices, 15 minutes for news). Without caching, every chat message asking about a stock would hit yfinance, adding 1–3 seconds of latency.

Dual-mode cache (Redis ↔ in-memory)

On Hugging Face Spaces, running a separate Redis container isn't practical. Aurum has a _InMemoryCache backed by cachetools.TTLCache that presents the same async interface. Setting USE_IN_MEMORY_CACHE=true in .env switches seamlessly — no code changes in the adapters or services that consume the cache.

# Cache abstraction — same interface for both backends
class _RedisCache:
    async def get(self, key: str) -> str | None: ...
    async def set(self, key: str, value: str, ttl: int) -> None: ...

class _InMemoryCache:  # HF fallback
    # One TTLCache per TTL bucket (60, 300, 900, 3600 seconds)
    async def get(self, key: str) -> str | None: ...
    async def set(self, key: str, value: str, ttl: int) -> None: ...

yfinance

Free real-time and historical market data

📈

yfinance

>=0.2.50
Yahoo Finance scraper — free, no API key required

yfinance wraps Yahoo Finance's undocumented API. It provides real-time quotes, historical OHLCV data, dividends, splits, and company fundamentals. No API key, no rate limit contract — though it can be blocked by Yahoo.

Why yfinance as the primary adapter?

Alpha Vantage has a 25-call/day free tier limit — useless for a demo with multiple users. Nasdaq Data Link requires sign-up. yfinance is free with no quotas and returns professional-quality data. The pluggable adapter registry means Aurum can fall through to Alpha Vantage if yfinance fails, and users can reorder adapter priority in Settings.

async def get_quote(self, symbol: str) -> Quote:

        lambda: yf.Ticker(symbol)

    return Quote(symbol=symbol, price=info["currentPrice"], ...)

Adapter Registry

Pluggable data sources with priority fallthrough

🔌

AdapterRegistry

src/adapters/registry.py
Strategy + fallthrough pattern for data adapters

The registry holds a priority-sorted list of market data adapters. When a quote is requested, it tries adapters in order, catching exceptions and falling through to the next one. Results are cached in Redis/in-memory to avoid redundant calls.

Design pattern: Strategy + Registry

Each adapter implements the same MarketDataAdapter ABC. The registry is the Strategy pattern — the caller doesn't know which adapter runs. The fallthrough behavior makes the system resilient: if yfinance is rate-limited or Yahoo blocks the IP, AlphaVantage automatically takes over. Users can reorder priority in the Settings page.

class AdapterRegistry:
    async def get_quote(self, symbol: str) -> Quote:
        cache_key = f"quote:{symbol}"
        if cached := await self._cache.get(cache_key):
            return Quote.model_validate_json(cached)  # cache hit
        for adapter in self._market_adapters:  # priority order
            try:
                result = await adapter.get_quote(symbol)
                await self._cache.set(cache_key, result.model_dump_json(), ttl=60)
                return result
            except Exception:
                continue  # try next adapter
        raise NoAdapterAvailable(symbol)

React + Vite + TypeScript

The frontend SPA — fast dev, type-safe, component-driven

⚛

React 18 + Vite 5 + TypeScript

react@18 · vite@5 · typescript@5
Component-based SPA with instant HMR

React organizes the UI into reusable components with local state. Vite replaces Webpack — it uses native ES modules during dev, giving sub-100ms hot module replacement. TypeScript catches type errors at build time, not in production.

Why React + Vite over Next.js or Streamlit?

Streamlit is Python-only but can't render Recharts-quality dashboards. Next.js adds RSC complexity unnecessary for a client-only SPA. React + Vite gives full control over the UI without Server Components overhead. The built ui/dist/ folder is served as static files from FastAPI — one fewer runtime to manage in production.

shadcn/ui + Tailwind CSS

Accessible components you own — styled with utility classes

🎨

shadcn/ui + Tailwind CSS v3

shadcn components · tailwindcss@3 · class-variance-authority
Copy-paste components built on Radix UI primitives

shadcn/ui isn't a component library you install — it's a collection of components you copy into ui/src/components/ui/ and own. Each component is built on Radix UI (accessible primitives) and styled with Tailwind. You can change any detail without fighting a library's CSS specificity.

Why shadcn over MUI / Chakra / Ant Design?

MUI, Chakra, and Ant Design inject their own CSS and have strict theming APIs. Customizing a MUI component deeply requires overriding sx props or styled-components. With shadcn, the component source lives in your codebase — edit it directly. Tailwind utility classes make it trivial to match Aurum's gold-accent dark theme.

Recharts

Composable React charts for portfolio and market data

📊

Recharts

recharts@2
D3-backed declarative charts for React

Recharts wraps D3.js in React components. Instead of writing SVG by hand, you compose chart components: <LineChart>, <PieChart>, <AreaChart>. Data flows in as props; Recharts handles the D3 scales, axes, and animations.

Why Recharts over Chart.js or Victory?

Chart.js uses Canvas (good performance, harder to customize). Victory is elegant but has a larger bundle. Recharts components are pure JSX — every axis, grid, tooltip, and legend is a React component you can style or replace. The portfolio's allocation pie, performance line, and goals projection chart are all Recharts.

TanStack Query

Server-state management — caching, background refetch, optimistic updates

🔄

TanStack Query v5

@tanstack/react-query@5
Async state management for server data

TanStack Query replaces useEffect + useState for API calls. It automatically deduplicates identical requests, caches responses with configurable stale times, refetches in the background, and handles loading/error states declaratively.

Why not Redux or Zustand?

Redux and Zustand manage client state (UI state, forms). Server data (portfolio holdings, market quotes) has different needs: cache invalidation, background updates, pagination, optimistic updates on mutation. TanStack Query was built specifically for this. Portfolio mutations use useMutation + queryClient.invalidateQueries to refresh automatically after a buy/sell.

Alembic

Database schema migrations — versioned, reversible

🔧

Alembic

>=1.14.0
SQLAlchemy's official migration tool

Alembic tracks schema changes as versioned Python files. alembic upgrade head runs all pending migrations. alembic downgrade -1 rolls back one version. Auto-generation compares your SQLAlchemy models against the live DB and generates the diff.

Why Alembic over Django migrations or manual SQL?

Django migrations are tied to Django's ORM. Alembic is the standard for SQLAlchemy projects. Manual SQL files don't track state — you have to remember what's been applied. Alembic maintains a alembic_version table in Postgres so it always knows the current schema version.

# Generate migration from model changes
alembic revision --autogenerate -m "add_goals_table"

# Apply all pending migrations (run on startup)
alembic upgrade head

# Rollback one version
alembic downgrade -1
