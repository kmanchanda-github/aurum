🔌 MCP Server — Model Context Protocol Integration

Aurum exposes its finance tools via the Model Context Protocol (MCP), allowing Claude Desktop and other MCP-compatible clients to query live market data, search the financial knowledge base, analyze portfolios, and run Monte Carlo projections — all from within a conversation.

The server ships with two transport modes controlled by a single environment variable. No code changes are required to switch between them.

Architecture

Claude Desktop (or any MCP client) │ │ stdio pipe ──────── default, local only │ HTTP + SSE ──────── set MCP_TRANSPORT=streamable-http ▼ mcp_server/server.py (FastMCP) ├── get_stock_quote / get_market_overview / get_stock_history │ └── src/adapters/market/yfinance_adapter.py (Yahoo Finance) │ ├── search_financial_news │ └── src/adapters/news/pygooglenews_adapter.py (Google News RSS) │ ├── search_knowledge_base / ask_finance_question │ └── src/rag/chroma_store.py → ChromaDB (51 articles) │ ├── analyze_portfolio │ └── yfinance_adapter × N symbols + local math │ └── calculate_goal_projection └── NumPy Monte Carlo (no external calls)
No Docker needed for 6 of 8 tools. Market data, portfolio analysis, news, and goal projections all run standalone. Only search_knowledge_base and ask_finance_question require ChromaDB (for the vector store).

Tools Reference

get_stock_quote internet
Real-time price, daily change, 52-week range, market cap, and volume for any ticker.
Args: symbol: str — e.g. "AAPL", "^GSPC" (S&P 500), "BTC-USD"
get_market_overview internet
S&P 500, NASDAQ Composite, and Dow Jones — current price and daily change. Good opening context for any market conversation.
Args: none
get_stock_history internet
OHLCV bars for any ticker over any period. Returns start/end price, total return, and up to 30 recent bars.
Args: symbol: str, period: str (1d / 5d / 1mo / 3mo / 6mo / 1y / 2y / 5y / 10y / ytd / max), interval: str (1d / 1wk / 1mo)
search_financial_news internet
Recent news articles via Google News RSS matching a query. Returns title, source, date, URL, and summary.
Args: query: str — e.g. "Federal Reserve interest rates", limit: int (1–10, default 5)
search_knowledge_base ChromaDB
Semantic search over 51 curated financial education articles. Returns relevant passages with source title and category. Supports category filtering to narrow results.
Args: query: str, categories: list[str] | None — filter to: investing / portfolio / market / goals / tax, k: int (1–10, default 5)
analyze_portfolio internet
Fetches live prices for a list of holdings and returns P&L, allocation percentages, and a portfolio summary. Pass holdings directly — no account login required.
Args: holdings: list[dict] — each item needs symbol, shares, cost_basis
Example: [{"symbol": "AAPL", "shares": 10, "cost_basis": 150.0}]
calculate_goal_projection offline
Monte Carlo simulation (1,000 paths by default) returning p10/p50/p90 outcomes and probability of reaching the target. Fully offline — pure NumPy, no network calls.
Args: monthly_contribution: float, current_amount: float, target_amount: float, years: int, risk_tolerance: str (conservative / moderate / aggressive), monte_carlo_runs: int (100–5000, default 1000)
ask_finance_question ChromaDB
RAG-grounded educational lookup: searches the knowledge base and returns the top-5 passages with source attribution. Use for conceptual questions about investing, taxes, or financial planning.
Args: question: str, categories: list[str] | None

Transport Modes

TransportHow to enableWho can connectAuth
stdio default Just run the server (no env var needed) Only Claude Desktop — private pipe on the same machine Not needed — no network port
streamable-http MCP_TRANSPORT=streamable-http Any HTTP client that can reach MCP_HOST:MCP_PORT Set MCP_API_KEY to require X-API-Key header

Auth Middleware — Why It Exists and How It Works

The auth middleware only matters for the streamable-http transport. Here is the distinction:

stdio transport (Claude Desktop): The MCP server is launched as a child process by Claude Desktop. Communication happens over stdin / stdout — a private OS pipe between two processes on the same machine. There is no TCP port, no network exposure, and no way for any external party to reach the server. Auth middleware is registered but never invoked.
streamable-http transport: The server binds a TCP port and accepts HTTP connections. If deployed to a VPS, exposed via a tunnel (ngrok, Cloudflare), or run on a shared network, anyone who can reach the port can call your tools — fetching market data, querying your knowledge base, or hammering the yfinance API on your behalf. The middleware prevents this by requiring a shared secret.

How the middleware works

When MCP_API_KEY is set, every incoming HTTP request must include the header:

X-API-Key: your-secret-key

If the header is missing or wrong, the middleware returns 401 Unauthorized before the request reaches any tool. If MCP_API_KEY is not set, the middleware passes everything through — suitable for local testing.

# No auth (local testing only)
MCP_TRANSPORT=streamable-http python3 -m mcp_server.server

# With auth (any remote deployment)
MCP_TRANSPORT=streamable-http \
MCP_API_KEY=your-secret-key \
python3 -m mcp_server.server

Installation

# Install Aurum with MCP extras
pip install -e ".[mcp]"

# Or with uv
uv pip install -e ".[mcp]"

This adds fastmcp>=2.0.0. All other dependencies (yfinance, numpy, chromadb, pygooglenews) are already part of the core Aurum install.

Claude Desktop Setup

Edit ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) or %APPDATA%\Claude\claude_desktop_config.json (Windows):

{
  "mcpServers": {
    "aurum-finance": {
      "command": "python3",
      "args": ["-m", "mcp_server.server"],
      "cwd": "/absolute/path/to/Aurum",
      "env": {
        "USE_IN_MEMORY_CACHE": "true"
      }
    }
  }
}

Restart Claude Desktop. The 8 Aurum tools appear in the tools panel.

With ChromaDB for RAG tools — add to the env block: "CHROMA_HOST": "localhost", "CHROMA_PORT": "8001" and run docker compose up chromadb -d && docker compose --profile seed up rag-ingest first.

Hugging Face Spaces — Claude Desktop via Remote MCP

When deployed to Hugging Face Spaces, the MCP server runs inside the existing FastAPI container — no extra container or port needed. It is mounted at /mcp on the same port 7860 that serves the web app.

Claude Desktop (your laptop) │ │ stdio ──► npx mcp-remote (local bridge process) │ │ │ │ HTTPS │ ▼ │ https://username-aurum.hf.space/mcp │ │ │ FastAPI app (port 7860) │ ├── /api/* main API │ ├── /mcp MCP server ◄── mounted here │ └── / React SPA │ └── MCP_ENABLED=true (set automatically in Dockerfile.hf) MCP_API_KEY=... (set in HF Space secrets)

Step 1 — Set secrets in your HF Space

In your HF Space settings → Repository secrets, add:

Secret nameValue
ANTHROPIC_API_KEY (or OpenAI)Your LLM API key
SECRET_KEYRandom 32-char string for JWT auth
MCP_API_KEYAny strong secret — used to authenticate Claude Desktop

MCP_ENABLED=true is already set in Dockerfile.hf — no extra config needed in the Space.

Step 2 — Configure Claude Desktop

Install mcp-remote if you haven't already: npm install -g mcp-remote

Then add to claude_desktop_config.json:

{
  "mcpServers": {
    "aurum-finance": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://YOUR_USERNAME-aurum.hf.space/mcp",
        "--header",
        "X-API-Key:YOUR_MCP_API_KEY"
      ]
    }
  }
}

Replace YOUR_USERNAME with your HF username and YOUR_MCP_API_KEY with the secret you set in Step 1. Restart Claude Desktop — the tools will appear.

How mcp-remote works: It runs as a local process that Claude Desktop talks to over stdio (the normal way). Internally it forwards every tool call as an HTTPS request to the remote MCP endpoint and streams the response back. Claude Desktop never needs to know about HTTP — it just sees a normal stdio MCP server.
Always set MCP_API_KEY when deploying to HF Spaces. HF Space URLs are public. Without the key, anyone on the internet can call your tools, fetch market data, and query your knowledge base without any rate limiting.

Local Docker — same pattern

When running docker compose up locally, MCP_ENABLED=true is set in docker-compose.yml and the MCP server is mounted at http://localhost:8000/mcp. Use the same Claude Desktop config with http://localhost:8000/mcp as the URL (no auth key needed for local).

Standalone Remote HTTP Deployment

If you want the MCP server as a separate process rather than mounted inside FastAPI:

# Start with auth on port 8002
MCP_TRANSPORT=streamable-http \
MCP_HOST=0.0.0.0 \
MCP_PORT=8002 \
MCP_API_KEY=your-secret-key \
python3 -m mcp_server.server

Add to docker-compose.yml for containerised deployment:

  mcp:
    build: .
    command: python3 -m mcp_server.server
    ports:
      - "8002:8002"
    environment:
      MCP_TRANSPORT: streamable-http
      MCP_HOST: "0.0.0.0"
      MCP_PORT: "8002"
      MCP_API_KEY: ${MCP_API_KEY:-}
      USE_IN_MEMORY_CACHE: "false"
      REDIS_URL: redis://redis:6379/0
      CHROMA_HOST: chroma
      CHROMA_PORT: "8000"
    depends_on: [redis, chroma]
